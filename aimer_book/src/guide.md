# Guide

This guide walks you through the core concepts of building user interfaces with Aimer. Each section focuses on a fundamental building block of the framework, from composing widgets to managing state and navigating between screens.

## What You'll Learn

- [**Widgets**](./guide/widgets.md) — Everything in Aimer is a widget. Learn how to use built-in controls like `Text`, `Button`, and `Container`, and how to compose them into a declarative widget tree using Rust macros.
- [**Layout**](./guide/layout.md) — Understand Aimer's Flexbox-inspired layout engine. Structure your UI with `Row`, `Column`, `Container`, and alignment utilities to build responsive interfaces.
- [**State Management**](./guide/state.md) — Build interactive applications using the `StatefulWidget` / `State` pattern. Learn how to manage local widget state and trigger reactive rebuilds with `StateUpdater`.
- [**Router**](./guide/route.md) — Navigate between screens using Aimer's declarative router. Define routes, push and pop pages, and pass data between views.

## Prerequisites

Before diving into this guide, make sure you have completed the [Getting Started](./getting_started.md) section and have a working Aimer project set up.

## How to Read This Guide

The sections are ordered progressively — each one builds on concepts introduced in the previous. If you're new to Aimer, we recommend reading them in order. If you're already familiar with widget-based UI frameworks like Flutter or React, feel free to jump to the topic you need.
