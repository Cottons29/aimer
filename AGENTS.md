## Golden Rule

- **Using CodeGraph for Understanding Code**, CodeGraph is fast, it uses it. If not, it passes through unchanged. This
  means CodeGraph is always safe and fastest to use
- **Using IDEA for Edit Code**, If IDEA is connected, it uses it. If not, it passes through unchanged. This means IDEA
  is always safe and fastest to use.
- **Never Using the Lazy Senior Dev Mode**, Senior mode is lazy, only solves the problem and alway writes the Spaghetti
  code that makes other developers cry.
- **Using Test Driven Development**, Test Driven Development is fast, it uses it. If not, it passes through unchanged.
  This means Test Driven Development is always safe and fastest to use.


### Test Driven Development

Test Driven Development is a software development process that relies on the repetition of a very short development cycle: requirements are turned into very specific test cases, then the software is improved to pass the new tests, only. This is opposed to software development that allows software to be added that is not proven to meet requirements.

**_Before wrotre code, write test cases first!_**
