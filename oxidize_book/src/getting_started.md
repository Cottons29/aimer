# Getting Started

Follow these steps to set up the **Oxidize** framework on your machine, create a new project, and run it.

## Prerequisites

Before starting, ensure you have the following installed on your system:

- **Rust**: Ensure you have the latest stable toolchain installed via [`rustup`](https://www.rust-lang.org/tools/install).
- **Platform-Specific Dependencies**:
  - **macOS / iOS**: Xcode and a Mac with Metal-compatible hardware.
  - **Android**: Android NDK.
  - **Web**: `wasm-pack` installed (`cargo install wasm-pack`).

## Installation

The `oxidize` CLI tool manages scaffolding, running, and building projects. Since Oxidize is evolving, you can install it directly from the repository.

```bash
# Clone the repository
git clone https://github.com/Cottons29/oxidize.git

# Enter the directory
cd oxidize

# Install the CLI tool
cargo install --path=./oxidize
```

## Creating a New Project

Once the CLI is installed, creating a new Oxidize application is easy:

```bash
oxidize new my_app
```

This will generate a new directory `my_app` with the basic project structure and necessary `Cargo.toml` dependencies.

## Running the App

Navigate to your newly created project directory and start the application using the CLI:

```bash
cd my_app
oxidize run
```

If everything is set up correctly, a window should open displaying a "Hello World!" message.

## Your First Widget Tree

Here's an example of the basic entry point for an Oxidize application, generated by the CLI:

```rust
#[oxidize::main]
pub fn start_app() {
    OxidizeApp::start(
        Container!(
            child: Text!(
                "Hello World!",
                text_align: text::TextAlign::MidCenter,
                text_style: TextStyle!(
                    color: Colors::Black,
                )
            )
        )
    );
}
```

The `#[oxidize::main]` macro marks the entry point, while `OxidizeApp::start()` mounts the widget tree. Notice how `Container!` and `Text!` macros make building a UI tree straightforward and readable!
